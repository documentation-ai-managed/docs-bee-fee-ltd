## Overview

Bee-Fee provides RESTful APIs for integrating casino games, sports betting, and poker services into your platform. You access all endpoints using HTTPS at `https://api.beefee.com/v1`. Authenticate requests with a Bearer token obtained via login.

<Columns cols={3}>
  <Card title="Games API" icon="gamepad-2" href="#games-endpoint">
    Retrieve casino games, slots, and table games.
  </Card>
  <Card title="Bets API" icon="target" href="#bets-endpoint">
    Place and manage sports bets.
  </Card>
  <Card title="Account API" icon="user" href="#account-endpoint">
    Check balances and user details.
  </Card>
</Columns>

<Callout kind="info">
  Rate limits apply: 100 requests per minute per token. Monitor `X-Rate-Limit-Remaining` headers.
</Callout>

## Authentication

Obtain a Bearer token by logging in with credentials. Include the token in the `Authorization` header for protected endpoints.

<Steps>
  <Step title="Request Token" icon="key">
    Send a POST request to the login endpoint.
  </Step>
  <Step title="Use Token" icon="lock">
    Add `Authorization: Bearer YOUR_TOKEN` to headers.
  </Step>
  <Step title="Refresh Token" icon="refresh-cw">
    Use refresh endpoint every 24 hours.
  </Step>
</Steps>

<Request tabs="cURL,JavaScript">
```bash
curl -X POST https://api.beefee.com/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_password"
  }'
```

```javascript
const response = await fetch('https://api.beefee.com/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'your_username',
    password: 'your_password'
  })
});
const data = await response.json();
```
</Request>

<Response tabs="200">
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 86400,
  "token_type": "Bearer"
}
```
</Response>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_TOKEN`.
</ParamField>

<ParamField header="Content-Type" param-type="string" required="true">
  Set to `application/json`.
</ParamField>

## Bets Endpoint

Place a sports bet.

<CodeGroup tabs="JavaScript,Python">
```javascript
const betData = {
  event_id: 'match_456',
  amount: 50.00,
  odds: 2.5
};
await fetch('https://api.beefee.com/v1/bets', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(betData)
});
```

```python
import requests
bet_data = {
    'event_id': 'match_456',
    'amount': 50.00,
    'odds': 2.5
}
response = requests.post(
    'https://api.beefee.com/v1/bets',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json=bet_data
)
```
</CodeGroup>

## Error Handling

Handle errors by checking HTTP status codes and `error` fields.

| Code | Type     | Description                  |
|------|----------|------------------------------|
| 400  | Bad Request | Invalid parameters.         |
| 401  | Unauthorized | Missing or invalid token.   |
| 429  | Rate Limited | Too many requests.          |
| 500  | Server Error | Internal server issue.      |

<Expandable title="Common Error Response" default-open="false">
```json
{
  "error": {
    "code": "INVALID_TOKEN",
    "message": "The provided token is invalid or expired."
  }
}
```
</Expandable>

<Callout kind="tip">
Always validate responses and implement retry logic with exponential backoff for `429` errors.
</Callout>

## Best Practices

- Use HTTPS only.
- Store tokens securely, never in client-side code.
- Paginate large responses with `limit` and `offset` parameters.
- Validate all inputs client-side before API calls.

Integrate these APIs to power your gaming platform with Bee-Fee's reliable services.
